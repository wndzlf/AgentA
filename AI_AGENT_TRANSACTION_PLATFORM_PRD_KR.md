# AI 에이전트 범용 매칭/거래 플랫폼 기획서 (v2.0)

작성일: 2026-02-14  
프로젝트 가칭: `Agent Match & Trade`

## 1. 제품 정의

하나의 앱 안에서 사용자가 카테고리를 선택하면,  
해당 카테고리 전용 AI 에이전트가 대화로 니즈를 수집하고,  
맞춤 리스트를 UI로 제시하며, 매칭/거래 확정까지 진행하는 플랫폼.

핵심은:

- `카테고리 테이블뷰`
- `셀별 고정 프롬프트`
- `AI 대화 + 실시간 리스트 제시`
- `매칭/거래 확정 플로우`

---

## 2. 네가 만들고 싶은 것 (요약)

- 완전 단일 기능 앱이 아니라 **범용 플랫폼**
- 친구 만들기, 소개팅, 사고팔기, 쇼핑, 명품 거래 등 카테고리를 셀로 분리
- 각 셀 진입 시 **미리 설계된 프롬프트와 폼 스키마**를 실행
- 유저는 AI와 대화하고, AI는 그 내용을 구조화해서 서버에 등록/탐색
- 다른 유저 데이터와 매칭하여 **카드형 리스트 UI**로 계속 추천

---

## 3. 핵심 UX

## 3.1 메인 화면

- 상단: 카테고리 검색 바
- 본문: 카테고리 테이블뷰 셀
  - 예: 소개팅, 친구 만들기, 축구 매칭, 명품 거래, 일반 중고

## 3.2 카테고리 진입 화면

- AI 채팅 영역
- 추천 리스트 영역(카드/이미지/요약정보)
- 필터/정렬(거리, 최근활동, 일치율 등)
- 액션 버튼(관심, 제안, 매칭 요청, 확정)

## 3.3 사용자 경험 원칙

- 사용자는 말로 입력
- AI는 요약/질문/추천을 반복
- 추천 리스트는 대화 내용이 바뀔 때마다 업데이트

---

## 4. 대표 시나리오: 소개팅

1. 사용자가 `소개팅` 카테고리 선택  
2. AI가 기본 정보와 선호를 질문  
3. 사용자가 성향/취향/선호 조건을 계속 입력  
4. AI가 조건을 구조화해서 서버에 반영  
5. AI가 조건에 맞는 이성 후보 리스트를 카드 UI로 제시  
6. 사용자가 피드백(더 적극적인 성격, 외모 스타일, 거리 등) 제공  
7. AI가 리스트 재정렬/재추천  
8. 상호 수락 시 매칭 확정

중요:

- 소개팅은 예시일 뿐, 다른 카테고리도 같은 엔진으로 동작

---

## 5. 시스템 구조

## 5.1 카테고리 팩 구조

각 카테고리는 아래 4개를 가짐:

- `Prompt Pack`: 대화 가이드, 질문 순서, 금지 정책
- `Schema Pack`: 저장할 필드 정의
- `Ranking Pack`: 추천 점수 계산식
- `Tool Pack`: 등록/검색/메시지/확정 액션

## 5.2 공통 엔진

모든 카테고리가 공통으로 사용하는 단계:

1. 의도 파악
2. 구조화 저장
3. 후보 검색
4. 순위화 및 리스트 제시
5. 대화 기반 재탐색
6. 상호 수락/거래 확정

---

## 6. 데이터 모델 (공통)

- `users`: 사용자 기본 정보
- `category_profiles`: 카테고리별 사용자 프로필
- `intents`: 대화로 추출된 현재 의도/조건
- `listings_or_candidates`: 검색 대상 데이터
- `matches`: 제안/요청/수락/거절 상태
- `conversations`: 대화 로그
- `actions_audit`: AI 실행 기록

---

## 7. 매칭/추천 로직

- 점수화 예시:
  - 선호 일치도
  - 금지 조건 충돌 여부
  - 활동성/응답성
  - 거리/시간/가격 조건
- 출력:
  - 카드 리스트 + 일치 이유 + 부족한 조건
- 대화 반영:
  - 사용자 피드백을 받을 때마다 점수 재계산

---

## 8. 안전/정책

- 카테고리별 금지행위 정책 필수
- 민감 카테고리(소개팅 등)는 상호 수락 전 개인정보 비공개
- AI 자동 액션은 정책 엔진 통과 후 실행
- 모든 주요 액션은 감사 로그 기록

---

## 9. 수익화

- 매칭/거래 성사 수수료
- 프리미엄 추천(우선 노출, 정교한 필터)
- 카테고리 운영자 도구 구독(B2B)
- 광고/제휴

---

## 10. MVP 범위

처음부터 모든 카테고리를 완성하지 않고,
공통 엔진 + 핵심 카테고리 2개로 시작:

- `소개팅`
- `일반 거래` 또는 `친구 만들기`

MVP 필수:

- 카테고리 테이블뷰 + 검색
- 셀별 프롬프트 적용
- 대화 기반 정보 수집
- 추천 리스트 UI
- 요청/수락/확정 상태 전환

---

## 11. 12주 실행 계획

### 1-2주

- 카테고리 팩 규격 정의
- 소개팅/거래 스키마 확정

### 3-5주

- iOS 기본 UI(테이블뷰, 채팅, 카드 리스트)
- 서버 API 및 저장 구조

### 6-8주

- 추천 점수/재추천 로직
- 요청/수락 플로우

### 9-10주

- 정책 엔진/감사 로그
- 베타 테스트

### 11-12주

- 성사율 개선
- 유료화 실험

---

## 12. KPI

- 추천 클릭률
- 요청 대비 수락률
- 매칭/거래 성사율
- 확정까지 평균 시간

- 유료 전환율

## 13. 결론

네가 만들고 싶은 서비스는:

`카테고리 테이블뷰 기반의 범용 AI 에이전트 매칭/거래 플랫폼`

즉,

- 사용자는 말한다
- AI는 수집/정리/추천/매칭한다
- UI는 리스트와 확정 동작을 명확하게 보여준다

이 구조로 가면 친구 만들기, 소개팅, 쇼핑, 명품 거래를 하나의 앱 안에서 확장 가능하다.
